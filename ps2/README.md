# doomgeneric — PlayStation 2 port

A PS2 port of [doomgeneric](https://github.com/ozkl/doomgeneric) built on the
[ps2dev](https://github.com/ps2dev/ps2dev) SDL2 toolchain.

## Design: upstream stays pristine

The upstream sources in `../doomgeneric` are kept **unmodified** so syncing with
ozkl is trivial. All PS2 code lives here in `ps2/`. Two upstream files are
*replaced* (compiled from here instead of upstream), and there are only **two
tiny `#ifdef __PS2__` hooks** in upstream core files:

| Upstream file | How PS2 differs |
|---|---|
| `doomgeneric_sdl.c` | replaced by `doomgeneric_ps2.c` (not compiled) |
| `w_file.c` | replaced by `w_file_ps2.c` (not compiled) |
| `d_main.c` | +1 guarded hook → `PS2_GetIWAD()` for IWAD selection |
| `i_sdlsound.c` | +1 guarded `printf` reporting the audio backend |

That's the entire upstream footprint (17 added lines, 0 deletions).

## Files here

| File | Purpose |
|---|---|
| `doomgeneric_ps2.c` | SDL2 video backend + `main()`; lazy SDL init so the boot log shows first |
| `ps2_bootscr.c` | GS text console for the boot log (redirects stdout via `_write`) |
| `ps2_audio.c` | loads `audsrv.irx` (SPU2) before SDL audio opens |
| `ps2_iwad.c` | `PS2_GetIWAD()`: pick IWAD from hostfs (controller menu) or embedded |
| `ps2_menu.c` | libpad controller menu drawn on the GS text screen |
| `w_file_ps2.c` | upstream `w_file.c` + a hook to serve the embedded WAD |
| `w_file_mem.c` | in-memory WAD backend (zero-copy from the embedded array) |
| `doom1_wad.c` | embedded shareware IWAD (generated by `bin2c` from `DOOM1.WAD`) |
| `audsrv_irx.c` | embedded `audsrv.irx` (generated by `bin2c`) |

## Building

Needs the `ps2dev` toolchain (`$PS2SDK`, `$GSKIT`). With the ps2dock image:

```sh
# Default: NO WAD embedded -> copyright-clean ELF (~7 MB).
docker run --rm -u $(id -u):$(id -g) -v <repo>:/work -w /work/ps2 ps2dock:local make

# Optional: embed the shareware IWAD as a built-in fallback (needs DOOM1.WAD here).
docker run --rm -u $(id -u):$(id -g) -v <repo>:/work -w /work/ps2 ps2dock:local make EMBED_WAD=1
```

`audsrv.irx` is copied from the SDK automatically.

## Copyright / WADs

Doom WADs are **not** part of this repository (`*.wad` is git-ignored) and you
must supply your own:

- The **shareware** `DOOM1.WAD` is freely redistributable (id Software permits
  it); `make EMBED_WAD=1` bakes it in for convenience.
- `DOOM.WAD`, `DOOM2.WAD`, etc. are **commercial** — do not commit, embed for
  distribution, or share them.
- The **default build embeds nothing**, so the distributable `doomgeneric.elf`
  contains no game data; it loads a WAD from hostfs at runtime. If none is
  present it prints a message asking you to add one.

## Running (PCSX2 or hardware)

Load `doomgeneric.elf`. It boots to a text log, then the demo.

- **WADs**: drop `DOOM.WAD` / `DOOM2.WAD` / etc. next to the ELF (hostfs). If
  several are found you get a controller menu; if one, it's used; if none, the
  embedded shareware WAD is used.
- **audsrv.irx**: loaded from `host:audsrv.irx` if present, else the embedded copy.
- **Sound**: SFX via SDL2_mixer + audsrv. Music (MUS/MIDI) is not expected to
  play (no MIDI synth on PS2).
